# Agent 学习（一）——如果世界上只有 LLM

> 系列真正的起点。我们不学概念——我们**自己发明 Agent**。

---

先立一个原则：以后我们不学概念，而是自己发明 Agent。

你会发现，Agent 的每一个组件都不是凭空出现的，而是因为**上一版 Agent 遇到了问题**。

这和你之前设计游戏框架的经历其实一模一样。你当时也不是一开始就设计 Entity、EventBus、ResourceManager、StateMachine，而是：游戏越来越复杂 → 发现问题 → 抽象出一个组件。

Agent 也是这个过程。

## 第三课：如果世界上只有 LLM，会发生什么？

今天，我们假设世界上还没有 Agent。只有一个 LLM。就像最早的 ChatGPT。

### 第一步：先定义 LLM 的能力

**Definition（定义）**

LLM（Large Language Model，大语言模型）本质上是一个：

> **根据已有信息，预测下一步最合适内容的推理系统。**

注意，我没有说它是"聊天机器人"。聊天只是它的一种应用。

如果站在 Agent 的角度，我们更关心的是：

```
输入（Input）
    ↓
理解（Understand）
    ↓
推理（Reason）
    ↓
输出（Output）
```

所以，LLM 更像一个**思考器（Reasoning Engine）**。

### 第二步：给它一个任务

任务：

> 帮我打开微信。

LLM 收到之后会发生什么？

它可能会想：

> 用户希望微信被打开。通常有几种方式：
>
> ① 点击桌面图标
> ② 从开始菜单搜索
> ③ 双击快捷方式

然后它回答：

> 你可以点击桌面的微信图标，或者按 Win 键搜索"微信"。

这里有一个关键点：**它完成了"思考"，但没有完成"目标"。**

为什么？因为它只能输出文字。

我们画个图：

```
          用户
            │
            ▼
      +-------------+
      |     LLM     |
      +-------------+
            │
      输出一句文字
            │
            ▼
       用户自己执行
```

注意最后一步。**真正完成任务的人是谁？用户。不是 LLM。**

### 那 LLM 到底有没有"行动能力"？

这是一个特别重要的问题。

答案是：**没有。**

很多人第一次听到会觉得奇怪。它不是已经"做了事情"吗？

没有。它只是：**告诉别人应该做什么。**

举个生活里的例子。

假设你有一个朋友，他特别聪明。你懂 Java，他懂所有知识。

你问："怎么修电脑？"

他说："第一步……第二步……第三步……"

他说得完全正确。但是**他一直坐在沙发上。**

电脑是谁修的？还是你。

所以：

> **知道怎么做 ≠ 能做到。**

### 我们来看几个例子

**例子一：查天气**

用户："帮我查东京天气。"

LLM："我建议你访问天气网站……"

为什么？因为它不知道今天的天气。它只能告诉你**怎么查**，而不是**帮你查**。

**例子二：整理桌面**

用户："帮我整理桌面。"

LLM："建议按照：图片、文档、视频、代码分类。"

很好。但是桌面整理了吗？**没有。**

**例子三：发微信**

用户："帮我给张三发一句：今晚七点见。"

LLM："可以发送：今晚七点见。"

消息发了吗？**没有。**

### 所以，LLM 最大的缺陷是什么？

很多人会回答：没有 Tool。

这个答案没错，但是还不够本质。

真正的缺陷其实是：

> **LLM 只能产生信息（Information），不能改变世界（World）。**

这是今天最重要的一句话。

我们来看两个东西。

**普通 ChatGPT：**

```
世界：微信关闭
LLM：建议点击微信。
世界：微信仍然关闭
```

有没有改变世界？**没有。**

**真正的 Agent：**

```
世界：微信关闭
Agent：点击微信。
世界：微信已经打开。
```

它**改变了世界**。
