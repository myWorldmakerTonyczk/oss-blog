# Agent 学习（十）——Memory 的三种形态

> 上一节留了三个问题。这一节先来看答案，然后展开 Memory 的完整分类。

## 先看你的回答

如果你对上一节三个问题的回答是这样的——

**问题 1**：因为 Agent 的思考是连续的，所以需要思考过程中的记忆。

**问题 2**：每一步思考都会反复执行上一轮的任务，反复扫描电脑上的文件。

**问题 3**：A 应该存长期记忆，因为这个信息长期有效；B 应该是短期记忆，因为这个任务完成后这个信息就没用了。

如果这是你的答案——你已经自然区分出了**长期记忆（Long-term Memory）**和**短期记忆（Short-term Memory）**。这其实就是工业 Agent 架构里的核心划分。

我们逐个看。

## 问题 1：为什么 Memory 不只是为了提高性能？

你的回答：因为 Agent 的思考是连续的，所以需要思考过程中的记忆。

**评分：10/10。**

这个回答比"缓存"层次高很多。关键词：**连续性（Continuity）。**

我们重新看 Agent。

**没有 Memory：**

```
任务1 → Agent 思考 → 行动 → 结束
下一次：
任务2 → Agent：我不知道之前发生了什么。
```

每次都是一个**新的** Agent。

**有 Memory：**

```
任务开始 → 读取过去状态 → 继续推理 → 行动 → 更新状态 → 继续
```

所以 Memory 的核心不是"让 Agent 快一点"，而是**让 Agent 的行为具有时间连续性。**

这个思想其实和游戏非常像。

做游戏引擎时：

```
玩家血量：HP = 100
受到攻击：HP = 70
下一帧：
如果状态丢失：HP 又变回 100
```

游戏是不是坏了？为什么？因为**世界状态没有保存。**

Agent 也是。

## 问题 2：没有 Memory，整理桌面会怎样？

你的回答：每一步思考都会反复执行上一轮任务，反复扫描文件。

正确。不过这里有一个细节。

严格来说，不是"获取之前的记忆"，而是**之前的状态不存在了**，所以只能重新观察环境。

例如：

**第一次：**

> Agent：扫描桌面。结果：1000 个文件。

**如果没有 Memory：**

> 下一轮：Agent：我不知道刚才有哪些文件。重新扫描。

**但是有 Memory：**

```json
{
  "scan_finished": true,
  "files": ["a.jpg", "b.pdf", "..."],
  "processed": 200
}
```

> Agent：继续处理剩余 800 个。

所以 Memory 保存的是：**Agent 对任务的认知状态。**

## 问题 3：A 和 B 应该怎么存？

你的回答：

- A（用户喜欢 Java）→ 长期记忆
- B（扫描文件）→ 短期记忆

**评分：10/10。完全正确。**

### 长期记忆（Long-term Memory）

特点：**长期有效。**

例如：

- 用户喜欢 Java
- 职业：后端开发
- 偏好：回答希望深入原理

什么时候用？**以后任何任务。**

例如——

用户："推荐学习路线。"

Agent 读取 Memory："用户懂 Java，想深入原理。" → 调整回答。

### 短期记忆（Short-term Memory）

特点：**当前任务有效。**

例如整理文件：

```json
{
  "task_id": "12345",
  "status": "扫描完成",
  "files_processed": 300,
  "files_remaining": 700
}
```

任务结束 → 删除。

架构：

```
              Agent
                |
        -----------------
        |               |
 Short Memory     Long Memory
 当前任务状态       用户长期信息
 临时              持久
```

## 现在引入一个重要概念：Working Memory（工作记忆）

你可能会发现，短期记忆和长期记忆之间还有一个东西。

比如整理文件：Memory 中有 1000 个文件——不能全部塞给 LLM。

那怎么办？需要**当前正在处理的信息。**

例如：

> 当前正在处理：`a.jpg` / `b.png` / `c.png`

这就是 **Working Memory（工作记忆）。**

类似人脑——

- 你知道自己是 Java 开发者 → **长期记忆**
- 但是现在你正在回答 Agent 问题 → 你脑子里当前关注的是 Tool、Memory、LLM → **工作记忆**

## 完整 Memory 架构

```
             Memory
                |
     --------------------
     |        |         |
Long-term  Short-term  Working
长期记忆   短期记忆   工作记忆

用户信息    任务状态     当前上下文
持久        临时         即时
```

## 停下来想一想

现在回顾一下 Agent 已经进化到哪里了：

**第一版**：LLM
→ 问题：不能行动。
→ 增加 Tool：**LLM + Tool**

**第二版**：LLM + Tool
→ 问题：无法保持连续状态。
→ 增加 Memory：**LLM + Tool + Memory**

```
             Memory
                ↑
                |
User → LLM → Tool → Environment
```

但是——现在还有一个巨大问题。我们下一节来发现它。
